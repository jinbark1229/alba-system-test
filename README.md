# 알바 시스템 (Alba Management System)

사장님과 알바생을 위한 통합 매장 및 근태 관리 시스템입니다. 

기존의 복잡한 엑셀 기반 시간표와 수기 출근부를 하나의 웹 애플리케이션으로 통합하여 효율적인 매장 운영을 지원합니다. 이 프로젝트는 포트폴리오 목적으로 구성되어 있으며, 외부인이 별도의 로그인 절차 없이(방문자 모드) 모든 기능을 체험할 수 있도록 설정되어 있습니다.

## 🚀 기술 스택

- **Frontend**: React 18, TypeScript, Vite
- **Routing**: React Router v7
- **Styling**: Tailwind CSS
- **Local Database**: LocalStorage (포트폴리오 전시용으로 브라우저 로컬 스토리지에 데이터 임시 저장)

---

## ✨ 핵심 기능

### 🧑‍🍳 알바생 (Worker)
- **근무 일지 작성**: 출/퇴근 및 휴게 시간을 기록하고 최근 작성 내역을 확인힙니다.
- **시간표 조회 (Schedule)**: 사장님이 업로드한 주별/월별 스케줄을 확인합니다 (내 스케줄 필터링 및 다중 매장 분리 기능).
- **근무 변경 게시판**: 시간표 댓글 기능을 통해 알바생 간 대타를 구하거나 일정을 조정할 수 있습니다.
- **급여 계산기**: 시급과 총 노동 시간을 기반으로 지급받을 예상 월급을 미리 확인합니다.
- **공지사항**: 매장의 주요 공지사항을 열람합니다.

### 👔 관리자 & 사장님 (Boss/Admin)
- **시간표 일괄 등록 (Excel/CSV)**: 직원의 이름과 근무 시간이 적힌 복잡한 형태의 엑셀 시간표를 드래그 앤 드롭으로 업로드하고 파싱합니다. (예: 8.5 시간 -> 8시간 30분 자동 치환)
- **근무 일지 내보내기 (ZIP Export)**: 전 직원의 근무 기록을 개별 CSV 파일 및 다중 ZIP으로 한 번에 내려받아 급여 대장이나 노무 자료로 활용합니다.
- **허용 이름 관리 (Whitelist)**: 사전에 등록된 알바생 이름과 고유 인증 코드를 통해서만 회원가입이 가능하도록 관리하여 외부인의 접근을 차단합니다.
- **공지사항 및 댓글 관리**: 매장 공지사항을 작성하고 직원들의 게시판 활동을 총체적으로 관리합니다.

## �️ 최근 업데이트 (포트폴리오 모드)

> 본 시스템은 포트폴리오 전시 모드로 작동 중입니다.

- **방문자 (Guest Admin) 시스템 도입**: 방문자가 `http://localhost:5173` 에 접근하면 번거로운 회원가입과 로그인을 거치지 않고 **'방문자(포트폴리오용) 관리자'** 로 자동 접속됩니다.
- **전체 메뉴 개방**: 보통 사장님과 알바생에게 각기 숨겨지던 8개의 메뉴 바(급여, 허용 이름 관리, 시간표 등)가 모두 통합 공개되어 기능 체험이 편안하게 가능합니다.

1. **저장소 클론 및 패키지 설치**
```bash
git clone https://github.com/yourusername/alba-system-test.git
cd alba-system-test
npm install
```

```bash

```

3. **브라우저 접속**
브라우저에서 `http://localhost:5173`에 접속합니다. (관리자 방문자 모드로 자동 접근됨)

## 🏛️ 보안 및 아키텍처 특징

- **Role-Based Access Control (RBAC)**: `worker`, `manager`, `boss`, `admin` 네 가지 권한으로 프론트엔드 컴포넌트 라우팅(`ProtectedRoute`) 보안이 강력하게 걸려 있습니다.
- **Persisted State (LocalStorage)**: 별도의 백엔드(Supabase 등) 연결 없이 브라우저의 로컬 스토리지(Local Storage)를 활용해 데이터를 저장합니다. 회원가입 정보, 근무 일지, 시간표 등 모든 상호작용은 사용자의 브라우저에 임시로 저장되며 새로 고침 후에도 데이터가 유지됩니다. (포트폴리오 시연 목적)
- **접근 권한 하드코딩 탈피**: 내부 UI 렌더링에 존재하던 하드코딩 권한 검사를 개선하여 포트폴리오용 게스트(`admin`)가 매장 관리 시나리오를 심도 있게 파악할 수 있도록 도왔습니다.
